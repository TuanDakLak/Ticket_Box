Description
Build the complete user authentication interface ecosystem and credential lifecycle management views. This task encompasses public-facing forms, account verification states, route guards, token expiration handling, and full client-side input validation integrated directly with the backend application endpoints.

Scope
In Scope:

Design responsive user interfaces for Sign In (/login) and Sign Up (/register) utilizing the project design system.

Implement account lifecycle verification views for account activation (/verify) and resending tokens (/resend-verification).

Implement recovery workflows including password initiation (/forgot-password), direct verification-linked updates (/reset-password), and account panel adjustments (/change-password).

Construct client-side state hooks utilizing the current user endpoint (/me) and secure context dispatchers.

Implement token rotation management via interceptors calling /refresh to silent-fetch Access Tokens using secure HTTP HttpOnly cookies or protected storage Refresh Tokens, along with session termination routing via /logout.

Enforce client-side schema validation using standard validation engines (e.g., matching email regex, character constraints, equality matches on password pairing).

Out of Scope:

Constructing server-side transactional mail distribution templates.

Building detailed layout wrappers for the high-level role management control panel views.

Steps
Scaffold UI component views for /login, /register, /verify, /forgot-password, /reset-password, and profile-level security setting forms.
Integrate form management frameworks to handle asynchronous execution states and direct inline input validation error messages.
Configure the HTTP service interface layers mapping directly to the following application authentication routes:
POST /auth/register
POST /auth/login
GET /auth/verify
POST /auth/resend-verification
GET /auth/me
POST /auth/forgot-password
POST /auth/change-password
POST /auth/reset-password
POST /auth/refresh
POST /auth/logout
Establish state hooks or context providers to retain token processing data and cache local user properties returned by the current identity route.
Create functional routing rules checking application tokens to block unauthenticated clients from visiting user profile or direct checkout screens.
Workflow Target

CI only

PR to develop

Staging deploy after merge

Release/deploy task
Notes
Token Rotation Dynamics: The /refresh operation must be hooked transparently inside client interceptors to recover from standard token timeout conditions smoothly before blocking active user layout sessions.
Submission Safety: All interface interactors must switch to locked/disabled loading configurations immediately upon trigger execution to prevent consecutive network endpoint traffic flooding (Request spamming).
Stack Adaptability: Technical tools, interface libraries, or state management frameworks can be freely adjusted or replaced by the team during initial development as long as they satisfy all verification endpoints and contract definitions.
